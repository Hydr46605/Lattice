## Added

- Added `ConfiguredUiException` for configured inventory authoring failures, including a stable `path()` accessor and contextual messages.
- Added configured inventory authoring documentation with a YAML-style menu example, fallback icons, configured actions, and resolver ownership guidance.

## Changed

- `ConfiguredInventoryUiCompiler` now reports invalid configured UI input with screen, page, button, icon, fallback, and action context.
- Configured UI resolver and text rendering failures are wrapped with configured UI paths so plugin authors can report actionable admin-facing errors.
- `Lattice.VERSION` is generated from the Gradle `latticeVersion` used by the build instead of being maintained as a separate Java literal.
- The GitHub Release workflow now supports stable `0.8.x` releases as latest releases, recovery switches for publication steps, and tag-pinned reruns for existing release updates.

## Fixed

- Maven Central `PUBLISHING` validation timeouts no longer prevent the workflow from continuing to mirror publication and GitHub Release creation.
- GitHub Release notes now only claim GitHub Packages mirror publication when the mirror publish step succeeds.
- Existing GitHub Release refreshes now refuse to upload assets built from a commit that does not match the release tag.
- Generated version sources are now wired into source jar task dependencies.

## Compatibility

- `ConfiguredUiException` extends `IllegalArgumentException`, so existing callers that catch authoring failures through the previous exception type continue to work.
- Existing configured inventory defaults are preserved, including default rows for non-positive row counts and default icon amount for non-positive amounts.
- `ConfiguredUiAction` remains a lightweight `type + data` declaration. Plugins still own action resolution through `ConfiguredUiActionResolver`.
