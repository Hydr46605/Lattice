## Added

- Added the `lattice-api` artifact as the first small shared API boundary for the 1.0 runtime split.
- Added a standalone Lattice host service that dependent plugins can discover through Paper services.
- Added shared JDBC pool management for the standalone runtime, with ref-counted HikariCP pools and storage diagnostics.

## Changed

- Prefer the standalone shared-runtime deployment model for new Paper/Folia plugins.
- Keep legacy shaded library usage available as isolated mode for self-contained plugin jars.
- Updated GitHub release automation to attach `lattice-api` artifacts.
- Added explicit Modrinth workflow checks for the project id and token.

## Compatibility

- Existing shaded plugins can keep using `LatticePaperPlugin` in isolated mode.
- Shared-runtime plugins should install the standalone `Lattice` plugin, compile against Lattice without relocation, and declare a hard Paper dependency with `join-classpath: true`.
