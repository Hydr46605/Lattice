## Added

- Added typed PacketEvents listener registration with Lattice-owned unregister handles, listener priority, packet event wrappers, and a safe API handle for version-specific access.
- Added command argument builders for custom parsers, completions, and greedy final string arguments.
- Added command help entries for flattened command-tree diagnostics and plugin-owned help commands.
- Added integration detail metadata, hook diagnostics, command-tree diagnostics, and isolated-mode active JDBC health reporting.
- Added README examples of public plugins built on Lattice: Tabular, Junction, and Onboard.

## Changed

- PacketEvents binding now reports API class/version details when available.
- Runtime diagnostics now include hooks and richer integration, command, and storage details.
- `DefaultIntegrationManager` closes `AutoCloseable` integration services during runtime shutdown.

## Fixed

- PacketEvents bootstrap now catches linkage failures in the same defensive style as PlaceholderAPI.
- Isolated storage diagnostics now report active JDBC connection health instead of only registered providers.

## Compatibility

- Existing `new CommandArgument<>(name, type, required)` calls continue to work.
- Existing `new CommandDiagnostics(name, aliases, description, permission)` calls continue to work.
- Existing `PacketEventsService#api()` callers continue to work; new code should prefer `apiHandle()` and typed listener registration.
